## What the agent needs before it can draft

### 1. `content/topic-queue.yaml`

Create this file with at least one entry in `queued` status. Each entry must have:

```yaml
- slug: example-slug
  title: "Example Post Title"
  angle: "The specific hook or angle for this post"
  keywords:
    - keyword one
    - keyword two
    - keyword three
  status: queued
```

The agent processes the **first** entry with `status: queued` and flips it to `status: drafted` after writing the post.

### 2. `src/content/blog/` directory with MDX support

The agent writes posts as `.mdx` files. The Vite/vite-react-ssg build needs to be able to:
- Import `.mdx` files with frontmatter parsed
- Render the blog listing page at `/blog`
- Render individual posts at `/blog/<slug>`

This requires adding a Vite MDX plugin (e.g., `@mdx-js/rollup` + `remark-frontmatter` + `remark-mdx-frontmatter`) and creating the blog route components.

### 3. `public/llms.txt`

Create this file with at minimum a `## Blog posts` section header. The agent appends a line per post to the top of that section.
